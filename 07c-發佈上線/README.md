# 07c 發佈上線（讓網頁真正上線）

> 🟡 **進階／應用關。** [07 推上 GitHub](../07-推上GitHub/) 是把**程式碼**放上雲端（作品集）；這一關更進一步——把網頁**部署成任何人都點得開的網站**，拿到一個公開網址。
>
> 用 Codex 的部署外掛幾乎全自動，很適合課後當回家作業挑戰。下面介紹兩個免費好用的平台：**Netlify** 與 **Vercel**，任選一個即可。

## GitHub vs 部署平台，差在哪？
| | [07 推上 GitHub](../07-推上GitHub/) | 07c 發佈上線（這頁） |
|---|---|---|
| 放上去的是 | 程式**原始碼** | 跑起來的**網站本身** |
| 別人打開看到 | 一堆檔案 | 你的網頁畫面 |
| 適合 | 收藏、版本管理 | 分享給學生／家長直接用 👈 |

兩者**互補**：常見做法是程式碼放 GitHub，網站掛 Netlify 或 Vercel。

## Netlify 還是 Vercel？選一個就好
| | Netlify | Vercel |
|---|---|---|
| 免費額度 | 個人／教學用綽綽有餘 | 個人／教學用綽綽有餘 |
| 最適合 | 純靜態網頁（HTML/CSS/JS） | 純靜態，也擅長 React/Next.js 等框架 |
| 給新手 | 都很好，**先學會一個**就夠 | 想之後玩前端框架可選這個 |

> 不用糾結——本課的小網頁兩個都跑得動。下面 **Netlify 為主示範**，Vercel 步驟幾乎一樣，附在後面。

---

## 方法 A：Netlify

### 步驟 1：安裝 Netlify 外掛程式
打開 Codex 左側邊欄的 **「外掛程式（Plugins）」** 面板，在 **Coding（編程）** 板塊找到 **Netlify**，點旁邊的 **加號（＋）** 或「安裝」加入。

### 步驟 2：帳號登入與授權
安裝過程中 Codex 會彈出 Netlify 登入頁。用 **GitHub 帳號** 或 **Email** 免費註冊登入，再點「授權」連上你的 Netlify。一次授權，之後不用再登。

### 步驟 3：下達部署指令
回到你的網頁專案對話框，直接用中文說：
```
請用 Netlify 幫我把這個網站部署上線。
```
Codex 會自動處理上傳與部署，你不用手動設定。

### 步驟 4：取得公開網址
幾分鐘後 Codex 會給你一個公網連結（像 `https://你的網站名.netlify.app`）。貼給任何人都打得開。🎉

---

## 方法 B：Vercel

跟 Netlify 幾乎一樣，只是換個平台。

### 步驟 1：安裝 Vercel 外掛程式
一樣在 Codex 的 **「外掛程式（Plugins）」** 面板，**Coding（編程）** 板塊找 **Vercel**，點 **加號（＋）** 安裝。

### 步驟 2：帳號登入與授權
Codex 會彈出 Vercel 登入頁。用 **GitHub 帳號** 免費註冊登入並授權即可（Vercel 與 GitHub 整合得很順）。

### 步驟 3：下達部署指令
在專案對話框直接說：
```
請用 Vercel 幫我把這個網站部署上線。
```

### 步驟 4：取得公開網址
完成後 Codex 給你一個 `https://你的專案名.vercel.app` 的公開連結，分享出去即可。

> 沒裝外掛也行：跟 Codex 說「幫我安裝並用 Vercel 部署這個資料夾」，它會帶你走（背後用的是 Vercel 官方 CLI `vercel`）。

## 卡關排解
- **找不到外掛？** 確認在「Coding／編程」板塊找；或用 Plugins 面板搜尋。也可直接跟 Codex 說「幫我安裝並用 Netlify／Vercel 部署」，由它帶你做。
- **部署後打開是空白？** 多半是進入點檔名不是 `index.html`。跟 Codex 說「把主頁命名為 index.html 再重新部署」。
- **多檔專案（專案四～六）也能部署嗎？** 可以。整個資料夾一起部署，平台會自動以 `index.html` 為首頁。
- **想換成自訂網域？** 兩個平台都支援，但屬進階設定；先用預設的 `.netlify.app` / `.vercel.app` 網址就很夠用了。
