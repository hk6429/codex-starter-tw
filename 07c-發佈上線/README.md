# 07c 發佈到 Netlify（讓網頁真正上線）

> 🟡 **進階／應用關。** [07 推上 GitHub](../07-推上GitHub/) 是把**程式碼**放上雲端（作品集）；這一關更進一步——把網頁**部署成任何人都點得開的網站**，拿到一個公開網址。
>
> 用的是 Codex 的 **Netlify 外掛程式（Plugin）**，幾乎全自動，很適合課後當回家作業挑戰。

## GitHub vs Netlify，差在哪？
| | [07 推上 GitHub](../07-推上GitHub/) | 07c 發佈 Netlify（這頁） |
|---|---|---|
| 放上去的是 | 程式**原始碼** | 跑起來的**網站本身** |
| 別人打開看到 | 一堆檔案 | 你的網頁畫面 |
| 適合 | 收藏、版本管理 | 分享給學生／家長直接用 👈 |

兩者**互補**：常見做法是程式碼放 GitHub，網站掛 Netlify。本頁先教最快的：用 Codex 外掛一鍵發佈。

---

## 步驟 1：安裝 Netlify 外掛程式
打開 Codex 左側邊欄的 **「外掛程式（Plugins）」** 面板，在 **Coding（編程）** 板塊中找到 **Netlify**，點旁邊的 **加號（＋）** 或「安裝」把它加入。

## 步驟 2：帳號登入與授權
安裝過程中，Codex 會自動彈出 Netlify 的登入頁。用 **GitHub 帳號** 或 **Email** 免費註冊並登入，再點「授權」，讓 Codex 正式連上你的 Netlify。

> 一次登入授權，之後發佈就不用再登。

## 步驟 3：下達部署指令
回到你的網頁專案對話框，直接用中文跟 Codex 說，例如：
```
請用 Netlify 幫我把這個網站部署上線。
```
Codex 會自動處理上傳與部署的所有事，你不用再手動設定。

## 步驟 4：取得公開網址
幾分鐘後，Codex 會在對話框給你一個 **公網可用的專屬連結**（像 `https://你的網站名.netlify.app`）。把這個網址貼給任何人，他們打開就能看到你的網站。🎉

## 卡關排解
- **找不到 Netlify 外掛？** 確認在「Coding／編程」板塊找；或到 Plugins 面板用搜尋。指令版（CLI）使用者可改跟 Codex 說「幫我安裝並用 Netlify 部署」，由它帶你做。
- **部署後打開是空白？** 多半是進入點檔名不是 `index.html`。跟 Codex 說「把主頁命名為 index.html 再重新部署」。
- **想換成自訂網域？** 那是 Netlify 進階設定，先用預設的 `.netlify.app` 網址就很夠用了。
